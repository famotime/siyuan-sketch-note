# Noteshelf 对标分析与商业化迭代路线图

> 日期：2026-05-24  
> 对象：`siyuan-sketch-note` 思源手写插件  
> 目标：参考 Noteshelf 3 Android 的成熟手写笔记体验，规划插件从“可用手写块”升级为“商业级手写笔记组件”的后续开发方向。

## 1. 资料来源与分析范围

本方案基于三类信息：

- 用户提供的 Noteshelf Android 截图。
- Noteshelf 官方 Android 页面与 Noteshelf 3 Android 页面，公开展示了真实笔刷/荧光笔、自定义工具栏、手写转文字、录音、形状、图片标注、富文本、模板、AI、手写搜索等能力。
- Google Play 的 Noteshelf 3 介绍，补充了 200+ 模板、Google Drive 同步、65 种语言手写识别、AI 摘要/翻译/解释等商业功能。

参考链接：

- <https://www.noteshelf.net/android.html>
- <https://www.noteshelf.net/noteshelf3-android.html>
- <https://play.google.com/store/apps/details?id=com.fluidtouch.noteshelf3>

## 2. Noteshelf 截图交互观察

截图展示的是横向上方工具栏 + 左侧浮动调色/笔宽面板 + 中央尺子工具的典型平板手写界面。

关键交互特征：

1. **双层顶部工具栏**：第一行偏文档级操作，包括返回、页面缩略图/网格、撤销、添加、禁用/模式切换、更多菜单；第二行偏绘制工具，包括笔、铅笔、橡皮、套索、形状/对象、尺子等。
2. **工具图标优先**：主操作全部是图标按钮，文本几乎不占用画布空间，适合高频手写。
3. **左侧浮动工具属性条**：颜色、当前色、添加颜色、笔粗等放在独立浮层中，点击工具后快速调参，不打断书写。
4. **尺子工具具备真实操控感**：尺子可旋转，有角度盘，线条沿尺边吸附，体现“物理文具隐喻”。
5. **画布为第一优先级**：工具栏半透明/深色，画布占据绝大部分区域，适合持续书写。
6. **工具状态清晰**：当前工具有灰色高亮背景，左侧当前笔宽/颜色也有显著状态。

这些设计说明成熟手写软件的核心不是“能画线”，而是让用户在长时间书写中低摩擦切换工具、精准控制线条、快速组织页面并可靠找回内容。

## 3. 当前插件能力基线

当前 `siyuan-sketch-note` 已具备：

- 在思源文档中插入手写块，并以图片缩略图回显。
- 全屏编辑器。
- Canvas 2D + Pointer Events 手写输入。
- 矢量笔迹数据持久化。
- 5 色画笔。
- 像素级橡皮擦。
- 撤销/重做。
- 白纸、方格纸模板。
- 自动增长画布。
- 高 DPI 适配。
- 明暗主题跟随思源。
- 自动保存与手动保存。

主要差距：

- 工具类型少：没有荧光笔、铅笔/钢笔差异、套索、形状、尺子、文本、图片、贴纸。
- 笔迹质量基础：当前为简单线段连接，缺少压感宽度、平滑、预测、防抖、笔刷纹理。
- 交互仍偏表单化：按钮带文字、工具栏占高、颜色与参数配置浅。
- 数据模型单一：只有 `Stroke[]`，缺少对象层、页面、附件、音频、OCR 索引等结构。
- 缺少商业级能力：搜索、手写识别、PDF/图片标注、导出、同步、模板库、快捷键/手势、性能保护。

## 4. 产品定位建议

插件不应完整复制 Noteshelf 的“独立笔记本系统”，因为思源已经负责文档、知识库、同步和块引用。更合理的定位是：

> 面向思源用户的高质量手写块与批注组件，让用户在文档中嵌入可编辑、可搜索、可导出、可长期维护的手写内容。

设计原则：

1. **尊重思源块模型**：手写内容仍以文档块中的图片/资产为入口，源数据存在插件数据中。
2. **优先提升书写体验**：延迟、笔迹、工具切换、误触防护优先级高于花哨功能。
3. **渐进商业化**：先做“像专业手写软件”的基础工具，再做 OCR、AI、同步等高级能力。
4. **保持可迁移**：所有专有数据应可导出 JSON + PNG/PDF，避免用户被锁死。

## 5. 迭代总览

| 阶段 | 目标 | 代表功能 | 优先级 |
| --- | --- | --- | --- |
| P0 | 夯实专业书写体验 | 工具栏重构、笔宽、压感、平滑、荧光笔、误触处理 | 最高 |
| P1 | 完善基础编辑工具 | 套索、对象选择、移动缩放、形状、尺子 | 高 |
| P2 | 丰富内容类型 | 文本框、图片导入/标注、贴纸、更多模板 | 高 |
| P3 | 页面与导出能力 | 多页、页面缩略图、PDF/PNG 导出、模板管理 | 中高 |
| P4 | 检索与智能能力 | OCR、手写搜索、转文字、AI 摘要 | 中 |
| P5 | 商业级可靠性 | 性能优化、数据迁移、备份恢复、诊断日志 | 持续 |

## 6. P0：专业书写体验与工具栏重构

### 6.1 图标化双层工具栏

目标：从当前“文字按钮工具条”升级为类似 Noteshelf 的高密度图标工具栏。

新增设计：

- 第一行：返回、保存状态、页面/画布概览、撤销、重做、添加、更多。
- 第二行：笔、荧光笔、橡皮、套索、形状、尺子、文本、图片。
- 桌面端显示 tooltip；触屏端长按显示名称。
- 工具选中态使用统一高亮背景。
- 小屏时工具栏横向滚动，不压缩画布。

开发要点：

- 新增 `ToolId`：`pen | highlighter | eraser | lasso | shape | ruler | text | image | hand`。
- 将工具栏拆为 `EditorTopBar.vue`、`ToolBar.vue`、`ToolOptionsPopover.vue`。
- 保留键盘快捷键：撤销/重做、保存、工具切换。

### 6.2 笔刷参数系统

目标：让每种工具拥有独立配置，而不是共享一个固定宽度。

新增能力：

- 笔宽滑杆：1-30 px。
- 透明度：尤其用于荧光笔。
- 常用颜色槽：支持新增、删除、最近使用。
- 每种工具记忆最近配置。
- 橡皮擦支持“像素橡皮”和“整笔橡皮”两种模式。

数据模型建议：

```ts
interface ToolPreset {
  tool: ToolId;
  color: string;
  width: number;
  opacity: number;
  mode?: string;
}
```

### 6.3 笔迹质量升级

目标：接近商业手写软件的流畅度和美观度。

新增能力：

- 使用 Pointer Events 的 `pressure` 控制线宽。
- 使用二次贝塞尔或 Catmull-Rom 曲线平滑笔迹。
- 添加最小点距过滤，减少抖动和数据量。
- 支持 Apple Pencil / S Pen / 触控笔的倾角字段预留。
- 添加“仅触控笔书写，手指拖动画布”的模式。

验收标准：

- 快速书写无明显折线感。
- 长笔记连续书写 5 分钟无明显卡顿。
- 缩略图与编辑器渲染效果一致。

## 7. P1：编辑工具、形状与尺子

### 7.1 套索选择

目标：让用户能选择、移动、缩放、删除局部笔迹。

功能：

- 自由套索路径选择。
- 框选模式作为可选项。
- 选中后显示边界框。
- 支持移动、缩放、删除、复制、改变颜色。

架构调整：

- `Stroke[]` 升级为 `SketchElement[]`。
- `StrokeElement` 作为其中一种元素。
- 每个元素有 `id`、`bounds`、`transform`、`zIndex`。

### 7.2 形状识别与形状工具

对标 Noteshelf 的“画完停顿自动转形状”。

功能：

- 长按笔尾或画完停顿 500ms 触发形状识别。
- 支持直线、箭头、矩形、圆/椭圆、三角形、多边形。
- 形状可二次编辑颜色、线宽、填充、透明度。

开发策略：

- 第一版先提供显式形状工具。
- 第二版再做手绘形状识别，降低误判风险。

### 7.3 尺子工具

目标：实现截图中的物理尺子隐喻。

功能：

- 可拖动、旋转的半透明尺子。
- 显示角度盘，例如 0°、45°、90°。
- 当笔迹靠近尺边时，输入点投影到尺边，生成直线。
- 支持水平/垂直/指定角度吸附。

开发要点：

- 尺子是编辑器 UI 状态，不一定持久化。
- 线条结果仍保存为普通 stroke 或 shape line。
- 需要处理触控冲突：双指移动尺子，单笔书写。

## 8. P2：内容类型扩展

### 8.1 文本框与富文本

功能：

- 点击插入文本框。
- 支持字体大小、颜色、粗体、斜体、列表。
- 文本框可移动、缩放。
- 与思源文本保持边界清晰：手写块内部文本不参与思源块编辑，但可导出。

### 8.2 图片导入与标注

功能：

- 从本地选择图片或粘贴剪贴板图片。
- 图片作为元素插入画布。
- 支持移动、缩放、裁剪、删除。
- 支持在图片上继续手写标注。

### 8.3 模板库

当前只有白纸和方格纸。建议扩展：

- 横线纸、点阵纸、康奈尔笔记、会议记录、待办清单、周计划、月计划。
- 模板支持颜色、间距、边距。
- 用户自定义模板：上传图片作为背景。

## 9. P3：页面、导出与文档级体验

### 9.1 多页手写块

当前模型是单个无限增长画布。商业软件通常有页概念。

建议：

- 保留“无限画布模式”作为轻量手写块。
- 新增“分页模式”，适合课堂笔记、会议纪要、PDF 导出。
- 左侧/顶部页面缩略图入口，对应截图中的网格按钮。

数据模型：

```ts
interface SketchDocument {
  version: 2;
  mode: "infinite" | "paged";
  pages: SketchPage[];
  settings: SketchDocumentSettings;
}
```

### 9.2 导出

功能：

- 当前页导出 PNG。
- 全部页面导出 PDF。
- 导出时可选择是否包含背景模板。
- 同时导出源数据 JSON，便于迁移和备份。

### 9.3 PDF 标注

独立笔记软件的 PDF 批注能力很重要，但在思源插件中应谨慎排期。

建议路线：

- 短期：支持图片/PDF 页面截图作为背景导入。
- 中期：导入 PDF 为多页背景。
- 长期：保留原 PDF 结构并导出可搜索批注 PDF。

## 10. P4：OCR、搜索与 AI

### 10.1 手写 OCR 与搜索

Noteshelf 的关键商业能力之一是手写搜索和手写转文字。

插件建议：

- 第一版：提供“识别当前页文字”按钮，结果保存到插件数据。
- 第二版：将 OCR 文本写入思源块属性或隐藏文本块，参与思源搜索。
- 第三版：支持跨手写块全文搜索，搜索结果跳转到对应手写块。

可选技术：

- 浏览器端 OCR：隐私好，但中文手写效果和体积需验证。
- 服务端/本地模型：效果更强，但部署复杂。
- 调用外部 AI/OCR API：效果最好，需配置密钥和隐私提示。

### 10.2 AI 辅助

适合商业版或高级功能：

- 总结当前手写页。
- 将手写内容整理成 Markdown。
- 翻译手写文本。
- 根据手写提纲生成思源文档。
- 对会议手写笔记生成待办事项。

必须注意：

- 默认不上传用户数据。
- 所有 AI 调用需要显式确认。
- 文档中说明服务商、数据范围和失败处理。

## 11. P5：性能、可靠性与工程治理

### 11.1 渲染性能

风险：笔迹数量增长后，全量重绘和 JSON 存储会变慢。

优化：

- 按页面/图层分块存储。
- 为 stroke 预计算 bounds。
- 局部重绘替代全量重绘。
- 大笔迹数据压缩，例如点坐标 delta 编码。
- 缩略图异步生成，避免阻塞保存。

### 11.2 数据迁移

从当前 `version: 1` 迁移到元素模型时必须提供兼容层。

要求：

- 老数据自动读取。
- 首次保存时迁移到新版本。
- 保留迁移失败回退。
- 提供“导出原始数据”诊断入口。

### 11.3 自动保存与冲突

增强：

- 保存队列串行化，避免并发覆盖。
- 页面关闭前强制 flush。
- 保存失败时保留本地临时副本。
- 显示最后保存时间。

## 12. 建议开发计划

### 里程碑 M1：专业工具栏与笔刷参数

目标：让界面接近专业手写软件，提升日常书写体验。

任务：

1. 拆分编辑器工具栏组件。
2. 引入图标化工具按钮和 tooltip。
3. 新增笔宽、透明度、颜色管理。
4. 新增荧光笔。
5. 新增触控笔/手指模式设置。
6. 对 README 增加新功能说明。

建议验收：

- 平板窄屏下工具栏不遮挡主要画布。
- 笔、荧光笔、橡皮配置互相独立。
- 保存后重新打开配置和笔迹效果一致。

### 里程碑 M2：笔迹引擎升级

目标：笔迹更顺滑、更接近真实书写。

任务：

1. 抽象 `renderStroke` 和输入采样模块。
2. 增加点距过滤。
3. 增加曲线平滑。
4. 增加压感宽度渲染。
5. 更新缩略图渲染逻辑，确保效果一致。
6. 补充画布引擎单元测试。

建议验收：

- 快速曲线不再明显折线。
- 低压轻写、高压重写宽度有差异。
- 橡皮擦仍能正确擦除压感笔迹。

### 里程碑 M3：套索与元素模型

目标：从“笔迹列表”升级为“可编辑对象画布”。

任务：

1. 新增 `SketchElement` 数据模型。
2. 编写 v1 stroke 到 v2 element 的迁移。
3. 实现元素 bounds 计算。
4. 实现套索选择。
5. 实现移动、删除、复制、改色。
6. 增加保存/读取兼容测试。

建议验收：

- 老手写块可打开并保存。
- 套索移动后导出的 PNG 与编辑器一致。
- 撤销/重做覆盖选择操作。

### 里程碑 M4：形状与尺子

目标：补齐学习、会议、图示类笔记常用工具。

任务：

1. 实现直线、箭头、矩形、圆形工具。
2. 实现形状选中与编辑。
3. 实现可拖拽旋转的尺子 UI。
4. 实现尺边吸附绘线。
5. 支持角度显示和常用角度吸附。

建议验收：

- 使用尺子可稳定画水平/垂直/斜线。
- 形状可保存、重开、导出。
- 双指移动尺子不误触书写。

### 里程碑 M5：图片、文本与模板库

目标：支持混合笔记。

任务：

1. 实现文本框元素。
2. 实现图片元素导入、粘贴、移动、缩放。
3. 新增横线纸、点阵纸、康奈尔模板。
4. 支持模板参数配置。
5. 支持自定义背景图片模板。

建议验收：

- 图片上可正常书写标注。
- 文本框导出清晰。
- 模板切换不破坏既有笔迹坐标。

### 里程碑 M6：搜索、OCR 与 AI

目标：让手写内容进入知识管理流程。

任务：

1. 设计 OCR 结果数据结构。
2. 实现当前页 OCR 入口。
3. 将识别结果写入思源块属性或伴随文本。
4. 实现手写块内搜索定位。
5. 预留 AI 摘要/整理接口。

建议验收：

- 识别文本可被思源搜索命中。
- 搜索结果能跳转并打开对应手写块。
- 未配置 OCR/AI 时功能入口有明确提示，不影响基础书写。

## 13. 技术债与风险

1. **Canvas 坐标模型**：当前固定逻辑宽度 800，后续需要支持缩放、分页和响应式显示，必须统一坐标系统。
2. **撤销栈成本**：当前 undo 保存完整 `Stroke[]` 快照，大数据会占内存。元素模型阶段应改为 command/history diff。
3. **橡皮擦语义**：像素橡皮依赖绘制顺序；元素编辑、导出、OCR 都要保留相同合成逻辑。
4. **OCR 成本与隐私**：中文手写 OCR 不宜贸然内置在线服务，应先做可选配置。
5. **思源集成边界**：插件不能假设可以完全接管文档编辑体验，应保持手写块独立、可关闭、可恢复。

## 14. 推荐优先级结论

短期最值得做的是 M1 + M2：工具栏、笔刷参数、荧光笔、笔迹平滑和压感。这些直接决定用户是否愿意长期书写。

中期做 M3 + M4：套索、元素模型、形状、尺子。这些是从“画板”走向“专业笔记工具”的分水岭。

长期做 M5 + M6：图片、文本、模板、OCR、AI。它们能扩大使用场景，但依赖稳定的数据模型和渲染引擎，不建议在基础书写体验成熟前优先投入。
